# 完成の定義（Definition of Done）

プロダクトバックログアイテム（PBI）が「完成」と見なされるためには、以下の全ての基準を満たす必要がある。

---

## 1. コードレビュー

- [ ] プルリクエストが作成され、少なくとも 1 名のレビューを受けていること
- [ ] レビュー指摘事項が全て解消されていること
- [ ] コーディング規約に準拠していること（TypeScript: フロントエンド / Python: バックエンド）

## 2. テスト

- [ ] ユニットテストが作成されていること
- [ ] 全てのユニットテストが通過していること
- [ ] Lambda 関数には正常系・異常系のテストケースが含まれていること

## 3. 構造化ログ

- [ ] 全ての Lambda 関数で JSON 形式の構造化ログが出力されること
- [ ] 必須フィールド（timestamp, level, service, endpoint, requestId, message）が含まれていること
- [ ] リクエスト完了ログに statusCode, durationMs が含まれていること
- [ ] エラーログに errorCode, errorType が含まれていること

## 4. ドキュメント

- [ ] 新規 API エンドポイントの仕様が記載されていること
- [ ] 環境変数や設定値が README に記載されていること
- [ ] 複雑なロジックにはコード内コメントが記載されていること

## 5. セキュリティ

- [ ] IAM ロール・ポリシーが最小権限の原則に従っていること
- [ ] シークレットや認証情報がハードコードされていないこと
- [ ] 環境変数で設定が管理されていること
- [ ] 依存パッケージに既知の脆弱性がないこと

## 6. インフラストラクチャ

- [ ] Terraform コードが記述されていること（該当する場合）
- [ ] `terraform plan` が正常に完了すること
- [ ] イミュータブルインフラストラクチャパターンに従っていること

## 7. 受入基準

- [ ] PBI に定義された全ての受入基準を満たしていること
- [ ] プロダクトオーナーが受入確認を完了していること
